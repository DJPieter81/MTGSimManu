## CRITICAL RULES — violations are bugs, not fixes

### NO HARDCODED CARD NAMES
Every check must use oracle text patterns, tags, CMC, power/toughness, or card types. Never `.name == "Card Name"`. Grep the codebase for `.name ==` patterns — if you are about to add one, STOP and find the oracle/tag equivalent.

### NO ARBITRARY CONSTANTS
Every number you write must be DERIVED from game state, not invented. Bad vs good examples:

BAD:  `if a_power >= 8: value *= 2.0`  — why 8? why 2.0?
GOOD: `value *= (a_power / max(b_power, 1))` — scales with actual power ratio

BAD:  `if snap.opp_creature_count >= 3: ev += 15.0`  — why 3? why 15?
GOOD: `ev += snap.opp_creature_count * attacker_val` — scales with board state

BAD:  `threshold = 0.7`  — magic number
GOOD: `threshold = opp_clock / (opp_clock + my_clock)` — derived from game pressure

The principle: if someone asks "why this number?", the answer must be "because the game state says so", never "I picked it".

Acceptable derived values:
- Ratios: a_power / b_toughness, my_creatures / opp_creatures
- Game-state fractions: damage / life, cards_in_hand / 7
- CMC-based: cmc * 1.5 (CMC is a real game property)
- Tag-driven: len(opp.creatures) * removal_value_per_target
- Profile-based: values from strategy_profile.py that are per-archetype (these ARE the tuning knobs)

### NO REGRESSIONS
After each fix, test BOTH the target matchup AND run a quick 4-deck spot check:
```
python3 iterate.py --matchup "DeckA" "DeckB" --n 10
python3 -c "from run_meta import run_matchup; [print(f'{d}: {run_matchup(\"Boros Energy\",d,3)[\"pct1\"]}%') for d in ['Affinity','Domain Zoo','Izzet Prowess','Eldrazi Tron']]"
```
Revert if ANY matchup moves >15pp in the wrong direction.

## The Loop (run until 6 hours elapsed or no HIGH outliers)

### Phase 1: Diagnose (every iteration)
```
python3 iterate.py --check --n 5
```

### Phase 2: Pick worst outlier, trace it
```
python3 run_meta.py --verbose "DeckA" "DeckB" -s 90000
python3 run_meta.py --verbose "DeckA" "DeckB" -s 90100
```
Read TWO traces. Look for the PATTERN, not one anecdote:
- Cards in hand never cast? -> scoring issue
- Blocks never happen? -> block eval issue
- Attacks never happen? -> attack threshold issue
- Wrong cards sideboarded? -> SB rule issue
- Combo pieces wasted individually? -> combo patience issue

### Phase 3: Fix (max 10 lines, 1 file, NO magic numbers)
Derive your fix from the game state. Use ratios, scaling, and proportional logic.

### Phase 4: Test + commit or revert
Accept if: target improved >=5pp AND spot-check has no regression >15pp.

### Phase 5: Log and continue
Append a line to iterate_log.json with results. Go to Phase 1.
