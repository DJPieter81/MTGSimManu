## THE LOOP

### Step 1: Check
python3 iterate.py --check --n 5

### Step 2: Pick an outlier
Pick the worst HIGH outlier. If you have already tried and failed on that outlier twice, SKIP it and pick the next one. Log skipped outliers.

### Step 3: Diagnose with TWO traces
python3 run_meta.py --verbose "DeckA" "DeckB" -s 90000
python3 run_meta.py --verbose "DeckA" "DeckB" -s 90100
Look for the PATTERN across both games, not one anecdote.

### Step 4: Make ONE fix
- Max 10 lines changed, max 1 file
- Every number derived from game state (ratios, fractions), not invented
- No card names — use oracle patterns, tags, types

### Step 5: Test
python3 iterate.py --matchup "DeckA" "DeckB" --n 10
Then spot-check:
python3 -c "from run_meta import run_matchup; [print(f'{d}: {run_matchup(\"Boros Energy\",d,3)[\"pct1\"]}%') for d in ['Affinity','Domain Zoo','Izzet Prowess','Eldrazi Tron']]"

Accept if: target improved >=5pp AND no spot-check regression >15pp.
Reject: git checkout -- <file>

### Step 6: Commit if accepted
git add <file> && git commit -m "Fix: <description>" && git pull --rebase origin main && git push origin main

### Step 7: Go to Step 1

## NO ARBITRARY CONSTANTS RULE
Every number must be DERIVED from game state:
- Ratios: a_power / b_toughness, my_creatures / opp_creatures
- Fractions: damage / life, cards_in_hand / 7
- CMC-based: cmc * modifier (CMC is real game data)
- Profile-based: values from strategy_profile.py per archetype
BAD: if x >= 8: y *= 2.0 (why 8? why 2.0?)
GOOD: y *= (x / max(baseline, 1)) (scales with actual ratio)
