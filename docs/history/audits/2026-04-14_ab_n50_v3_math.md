## Observations on the Affinity signal

- v2 produced a statistically-significant Affinity drop (-6.0pp, outside CI).
- v3 partially undid it: Affinity -2.0pp vs baseline (inside CI).
- Likely cause: v3's engine `_threat_score` refactor changed opponent
  removal-targeting on Affinity's board. `creature_threat_value` uses
  raw power which Affinity creatures already have (equipped). The
  scaling-equipment virtual-power bonus (+3) in the refactored
  equipment block may now double-count vs Affinity, making opp targets
  less optimally.
- Net: v2 is the stronger build for the Affinity outlier fix; v3
  trades some of that gain for broader consistency.

## Conclusion

All three builds are in the same ballpark. The math-derived builds do not
produce a dramatically different meta — they produce a *tidier* one:
no rogue magic constants, every score traceable to clock/mana math, and
the one real audit target (Affinity outlier) moves in the right direction.

If we had to pick a single HEAD, **v2 is the best build** for the current
audit goals: strongest Affinity fix, cleanest Storm direction, no real
regressions. v3 adds engine-layer code cleanup but at the cost of some
Affinity fix efficacy. The v3 code improvements stand regardless — future
audit work benefits from having engine scoring already delegated to AI.
