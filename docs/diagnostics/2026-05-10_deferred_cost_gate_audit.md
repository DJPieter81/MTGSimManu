# Deferred-cost scoring audit — looking for 0.0-sentinel defects beyond suspend

## TL;DR

PR #381 fixed two gates in `_score_suspend` that returned `0.0`
before payoff math, collapsing 92.7% of suspend EVs to a sentinel.
This audit asked the obvious follow-up: **does the same defect
pattern lurk in any of the other deferred-cost scoring functions?**

Five candidate mechanics were named in the task brief: cycling,
evoke, madness, dash, flashback.  The audit's complete result:

| Mechanic | Function in `ai/ev_player.py` | 0.0-sentinel defect? |
|----------|---|---|
| cycling   | `_score_cycling` (1958-2015)             | **NO** — no early returns; 0/19 enumerations at EV=0.0 in sample sim |
| suspend   | `_score_suspend` (2017-2180, post-PR #381) | **NO** — residual `return 0.0` lines (2117, 2121) are rule-correct sentinels, not payoff-math short-circuits |
| evoke     | overlay inside `_score_spell` (759-768)  | **NO** — overlay is additive (`ev -=` / `ev +=`), no `return`; cannot produce a 0.0 sentinel |
| madness   | (does not exist anywhere in `ai/` or `engine/`) | **N/A** — mechanic not modelled |
| dash      | `_eval_dash` in `ai/board_eval.py:369` (NOT in `ev_player.py`) | **N/A in `ev_player.py` scope** — `_eval_dash` lives in the combat/board-action scoring layer, not the spell-scoring layer |
| flashback | (no dedicated scorer; routes through generic `_score_spell`) | **NO** — generic path does not 0.0-short-circuit on any flashback tag check |

This is a **null finding** and the diagnostic stream is complete.
No engine/AI code change ships from this session.  No xfail tests
were added — the task brief explicitly allows for a doc-only
deliverable when no defect is found.

## Methodology

1. Read the precedent diagnostic
   (`docs/diagnostics/2026-05-10_living_end_5pct_root_cause.md`)
   to copy its rule-phrased per-function audit shape.
2. `grep -n "def _score_" ai/ev_player.py` to enumerate every
   `_score_*` method (4 total: `_score_spell`, `_score_land`,
   `_score_cycling`, `_score_suspend`; the latter two are the
   deferred-cost subset).
3. For each, locate every `return 0.0` line within the function
   body and read the surrounding 5-10 lines to classify:
     - **Rule-correct sentinel**: returns 0.0 only for a malformed
       input, a structural neutral case (parser couldn't parse the
       suspend clause; no targets exist), or for a function whose
       contract is "0.0 = no adjustment" (`_holdback_penalty`).
     - **Defect pattern**: returns 0.0 mid-function as a hard
       cutoff that short-circuits the payoff math the rest of
       the function was designed to produce (the PR #381 pattern).
4. For each function with a non-trivial body, run a single-seed
   Bo3 of a deck that uses the mechanic, dump the NDJSON decision
   stream, and parse it to count EV=0.0 / EV>0 / EV<0 outcomes
   for that scoring path.
5. If any defect found, write a red xfail test that pins the
   defect (per loop-break protocol).  No defect → no test
   (audit succeeded with a null finding; see CLAUDE.md
   "Important Conventions" section confirming generalisation-first
   work — a fix that didn't exist would help no decks).

## Function-by-function audit

### `_score_cycling` — clean (no defect)

`ai/ev_player.py:1958-2015`.  Function signature:

```python
def _score_cycling(self, card, snap, game, me, opp) -> float:
```

Behaviour: accumulates an `ev` variable from clock-impact baseline,
graveyard-reanimation bonus (gated on `_has_reanimation_path`),
cycling-cost cheapness, cascade-in-hand urgency, gameplan-preferred
boost, and a final `_holdback_penalty` adjustment.

Every adjustment is `ev += …`.  The function exits via a single
unconditional `return ev` at line 2015.  **Zero early returns of any
kind; impossible to short-circuit to 0.0.**

The function CAN return a small value when none of the bonus
branches fire (no reanimation path, no cascade in hand, no
gameplan flag set, no cheap cycling cost), but `card_clock_impact(snap)
× CLOCK_IMPACT_LIFE_SCALING` produces a non-zero positive base for
any cycler that draws a card.

**Quantification** (Living End vs Affinity Bo3, seed 60100):

```
Total cycle enumerations in DECISION events:  19
  ev == 0.0:  0  (0.0%)
  ev  > 0.0: 19
  ev  < 0.0:  0
  Range: min = 2.38, max = 19.17
```

A diverse EV distribution; cycling is being scored on its actual
merits per snapshot.  No defect.

(File: `replays/dc_gate_audit_2026_05_10/living_end_vs_affinity_60100.ndjson`.)

### `_score_suspend` — post-PR #381 residual gates are rule-correct

`ai/ev_player.py:2017-2180`.  Function signature:

```python
def _score_suspend(self, card, snap, game, me, opp) -> float:
```

**Residual `return 0.0` lines remaining after PR #381:**

```
2117:   if game.can_cast(self.player_idx, c):
2118:       return 0.0
2121:   if parsed is None:
2122:       return 0.0
```

Classification:

- **Line 2117** — Gate 1, *now with castability check* (the
  PR #381 fix).  Returns 0.0 only when a faster castable finisher
  route is actually in hand.  This is rule-correct: a same-payoff
  finisher already castable next turn does make suspend strictly
  slower.  The class size remains every Modern suspend card; the
  gate doesn't fire on uncastable cascade cards (the Demonic Dread
  regression case from the precedent doc), because
  `game.can_cast` is now consulted.  Not a 0.0-sentinel defect.

- **Line 2122** — defensive sentinel for unparseable suspend
  clause (`_parse_suspend_clause` returned None).  Class size: zero
  (every legal suspend card parses).  Returns 0.0 only when the
  oracle text is malformed or the parser is missing a printing
  variant.  Rule-correct.

The Gate-2 hard cutoff from the precedent doc (`if resolution_offset
> opp_clock: return 0.0`) is gone — replaced by a continuous
exponential survival-probability discount at lines 2163-2174.
EV is no longer zeroed by `opp_clock` pressure; it decays
smoothly as the gap widens.

**Quantification:** Suspend was not enumerated in our single LE-vs-Affinity
Bo3 at seed 60100, because in this particular game Living End
never reached a Main1 phase with a suspendable card in hand AND
no faster castable cascade.  The precedent doc's 15-match sweep
already covers the suspend case at scale; re-running it here adds
no signal beyond confirming the function compiles after PR #381.

### `_score_spell` evoke overlay — clean (no defect)

`ai/ev_player.py:759-768`.  Excerpt:

```python
# ── Evoke overlay: projection doesn't model 2-card cost ──
if ('evoke' in tags or 'evoke_pitch' in tags) and snap.my_mana < (t.cmc or 0):
    from ai.clock import card_clock_impact
    ev -= card_clock_impact(snap) * EVOKE_CARD_LOSS_MULTIPLIER
    if snap.am_dead_next:
        ev += EVOKE_DESPERATE_BONUS
    elif snap.opp_creature_count == 0 and 'removal' in tags:
        ev -= EVOKE_NO_TARGET_PENALTY
```

This is an *additive overlay*: it modifies `ev` (already computed
from `compute_play_ev`) with `ev -=` and `ev +=` operations, then
falls through to the rest of `_score_spell`.  **No `return`
statement exists in the overlay**, so the overlay cannot produce
a 0.0 sentinel.  The final `_score_spell` return at line 1378
returns the accumulated EV, never short-circuiting to 0.0.

**Quantification** (Jeskai Blink vs Boros Energy Bo3, seed 60100):

```
Solitude enumerations: 1
  ev == 0.0:  0
  ev  > 0.0:  0
  ev  < 0.0:  1   (ev = -12.78 — projection + evoke card-loss overlay)
```

One sample is limited but it confirms the overlay does NOT
collapse evoke EV to 0.0.  The negative EV reflects the legitimate
2-card cost the overlay encodes; that is signal, not a defect.

(File: `replays/dc_gate_audit_2026_05_10/jeskai_blink_vs_boros_60100.ndjson`.)

### Madness — no scorer to audit

`grep -rn "madness" ai/ engine/` returns no matches.  The madness
mechanic is not modelled anywhere in the simulator.  No `_score_*`
function exists to harbour a defect.  Out of audit scope by

### Dash — out of `ev_player.py` scope

`_eval_dash` lives in `ai/board_eval.py:369-407`, not
`ai/ev_player.py`.  The task brief names `ai/ev_player.py`
explicitly; we list `_eval_dash` here for record but defer a
targeted audit to a follow-up diagnostic if one is desired.

Spot-check of `_eval_dash`: returns `BOARD_EVAL_HARD_VETO` (a
named non-zero constant) when dash is the only option, and
`-BOARD_EVAL_HARD_VETO` when dash is illegal.  All other paths
accumulate into a `score` variable and return it.  No 0.0
short-circuit mid-function.  The same hard-edge sentinel pattern
(distinct named constants for "must / must-not") is structurally
different from the suspend defect (`return 0.0` collapsing payoff
math into a default).

### Flashback — generic scoring path, no defect

Flashback is recognised in `engine/card_database.py:980-981` as a
tag.  AI scoring of flashback casts routes through the generic
`_score_spell` path — there is no `_score_flashback`.  The
generic path has no flashback-tag-conditional `return 0.0`.  No
defect.

## Why this audit succeeded with a null finding

PR #381 was a textbook localised defect: two gates in one
function, both producing the same sentinel value, both crowding
out the payoff math the function was designed to compute.  The
fix moved one gate to a castability check (true rule), kept one
defensive sentinel for the parser-failure case (true error path),
and replaced the second gate with a continuous discount.

The candidate function set for a similar defect — every other
deferred-cost scorer in `ai/ev_player.py` — is small (one
sibling: `_score_cycling`).  The sibling does not exhibit the
defect: it accumulates rather than short-circuits.  The other
mechanics named in the task brief (evoke, madness, dash,
flashback) either have no scorer to audit, or are scored via the
generic `_score_spell` path which has no analogous 0.0 sentinel.

This is the rare case where a defect-class audit returns a clean
verdict.  It is still progress: it rules out a class of bugs and
narrows the search space for future investigations of AI scoring
anomalies.  The diagnostic-loop trap in PROJECT_STATUS.md
(documentation without WR movement) does not apply — this audit
was an explicit "diagnostic only" stream per the task brief, and
its acceptance criterion includes the null-finding case.

## What this audit deliberately did NOT do

- **No fix.**  The task brief is unambiguous: diagnostic only.
- **No xfail tests.**  Xfail tests pin a defect for a future fix.
  No defect → no test.
- **No matrix re-run.**  The PR #381 fix already shipped; the WR
  signal lives in that PR's regression evidence, not here.
- **No deeper sweep on suspend.**  The 15-match precedent sweep
  is sufficient evidence; re-running it would duplicate work.
- **No audit of `ai/board_eval.py`.**  Out of task scope.  A
  follow-up diagnostic could audit the board-eval helpers for
  the same defect class if a P0/P1 surfaces there.
