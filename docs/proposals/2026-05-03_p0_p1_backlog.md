## P0 — open

### P0-A. Counterspell consistency
- **Status:** active, primary
- **Evidence:** Per-replay counter hit rate varies 14–100% across same matchup. Audit:
  - `dimir_midrange_vs_boros_energy_s60005.txt`: 8 cast, 5 hit (62%).
  - `azorius_control_vs_boros_energy_s60200.txt`: 7 cast, 1 hit (14%).
  - `dimir_vs_affinity_bo3.txt` + `_v2.txt`: 4–5 cast, 1 hit (~20%).
- **Hypothesis:** `ai/response.py` counter-decision rates Sojourner's Companion / Thought Monitor cast (Affinity-discount, free) as not-worth-countering. Threshold inverts when threat is moderate (response_value < 5) but board impact is high.
- **Failing-test rule:** `Counter fires on cost-reduced threat when held with mana available` — opp casts Sojourner's Companion (CMC 7, paid 0), Stubborn Denial in hand, ferocious live → counter must fire.
- **Hard-fail signal in WR data:** any deck with Counterspell loses to Affinity ≥ 80% of the time.
- **Effort:** medium (1–2 sessions). One agent attempt salvaged by hand-fix in failing test scaffolding.

### P0-B. Removal target inversion vs Affinity-style boards
- **PROJECT_STATUS bug:** §7 P1 #5 + Failed Attempt audit bug A.
- **Status:** active, primary
- **Evidence:** Live sim scores Springleaf Drum at 1.333, Memnite at 1.150 — picks Drum. Isolated repro of the same battlefield state correctly scores Memnite at 1.15 vs Drum at 1.00. **State drift between repro and live sim.** `_threat_score` accumulates something it shouldn't.
- **Failing-test rule:** `_threat_score(Memnite) > _threat_score(mana_rock)` regardless of prior turn history. Test sets up identical board via two paths (fresh build vs T5 of game with N prior turns) and asserts equal scores.
- **Effort:** medium-high. State-drift bugs are harder to localize. One agent attempt produced a `partial_snap.{my,opp}_artifact_count += 1` patch that may be the right idea but the test was unrunnable (`CardInstance.__init__` API misuse). Salvage that diff, write a working test.

### P0-C. Affinity dominance
- **PROJECT_STATUS bug #12** (was 93%, now 84.9% after this session's matrix).
- **Status:** active, primary
- **Note:** Engine math is rules-correct (Construct Token sizing, Plating buff, Sojourner's 4/4 base, Scion of Draco domain discount — all verified in audit). The fix is on opponents' side, not Affinity's.
- **Likely auto-fixed by P0-A + P0-B.** Schedule: re-run the matrix after P0-A and P0-B land; if Affinity drops to <70% WR, this closes. Otherwise diagnose further.

## P1 — open

### P1-1. Chalice/Stax undervalued — threat-gated EV
- **PROJECT_STATUS** §7 P1 #4. Failed Attempt §: `ai/stax_ev.py` built and tested (13 unit tests pass) but not wired — caused WST v1 36% → 32% regression because stax EV competed with Counterspell on T2 and lost the concrete answer.
- **Fix path:** stax EV only fires when (a) no opp threat needs this turn's mana, OR (b) AI would otherwise pass.
- **Effort:** medium. Module exists; threat-gate is the new condition.

### P1-2. Amulet Titan low WR (~23%, expected ~45%)
- **PROJECT_STATUS** §7 P1 #1. `ai/ev_player.py:_score_land()` — Arboreal Grazer not prioritising bounce lands; mana loop not modelled.
- **Effort:** high. Amulet's value is multiplicative across the game, hard to capture in a per-play scoring layer.

## Workflow

Each bug gets a `claude/fix-<bug-name>` branch with the failing test in the
same commit as the fix (per CLAUDE.md ABSTRACTION CONTRACT). After merge,
re-run the matrix and update WR deltas in this doc. When a bug closes, add
`status: resolved` and `superseded_by: <commit-sha>`.

The recurring Cowork task `weekly-fix-attempt` (see Scheduled tasks
documentation) picks the highest-priority open bug from this list and
attempts a fix branch each Friday.
